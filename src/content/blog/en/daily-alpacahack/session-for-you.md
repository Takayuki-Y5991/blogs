# Session for You

WriteUp for the `Message For You` challenge from 2026/04/02.

## Overview

A challenge where the flag is stored in Flask's client-side session (a signed cookie),
and you need to decode and read it.

## Code Walkthrough

The server-side code is as follows.

```python
from flask import Flask, session
import os
import secrets

FLAG = os.environ.get("FLAG", "Alpaca{**REDACTED**}")

app = Flask(__name__)
app.secret_key = secrets.token_hex(32)

MESSAGE = f"""
Roses are red,
Violets are blue,
I've hidden a flag
In a session for you: {FLAG}
""".strip()

HTML = """
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Message for You!</title>
</head>
<body>
    <p>I've got a message for you.</p>
    <p>It's hidden somewhere around here...</p>
</body>
</html>
""".strip()

@app.get("/")
def index():
    session["message"] = MESSAGE
    return HTML

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=3000, debug=True)
```

Accessing `/` stores the `MESSAGE` containing the flag in the session and returns it as a cookie.
The HTML contains no hint of the flag — you need to read the contents of the session cookie.

`app.secret_key` is randomly generated via `secrets.token_hex(32)`, so forging the signature is not possible.
However, Flask session cookies are only signed, not encrypted, meaning anyone can read the payload.

### 1. Understanding the Flask Session Cookie Structure

Flask session cookies are generated by the `itsdangerous` library and have the following structure:

```
<payload>.<timestamp>.<signature>
```

- `payload`: JSON compressed with zlib and encoded in base64url
- `timestamp`: Issuance time
- `signature`: HMAC signature (requires `secret_key`)

When the payload exceeds a certain size, zlib compression is applied and the cookie value is prefixed with `.`.

```
.eJyrVsp...  →  starts with . → zlib compressed
eJyrVsp...   →  no prefix    → not compressed
```

### 2. Retrieving the Session Cookie with curl

```bash
curl -v http://localhost:3000/
```

Grab the session value from the `Set-Cookie` header in the response.

```
Set-Cookie: session=.eJyrVspNLS5OTE9VslIK...; HttpOnly; Path=/
```

### 3. Decoding the Payload

Extract the second part (the payload) split by `.`, convert from base64url to base64, then decompress with zlib.

## Background Knowledge

#### How Flask Session Cookies Work

Flask stores session data on the client side (in a cookie) by default, rather than on the server.
The data is signed but not encrypted, so the payload can be read without the `secret_key`.

#### Detecting zlib Compression

`itsdangerous` automatically applies zlib compression when the payload exceeds a certain size, and prefixes the cookie value with `.`.
You can tell whether compression was applied by checking for the leading `.`.

#### base64url vs base64

Standard base64 uses `+` and `/`, but base64url replaces them with `-` and `_` for safe embedding in URLs.
You need to convert with `tr '_-' '/+'` before decoding.

#### zlib Decompression in Python

```python
import sys, zlib
# Read binary from pipe, decompress with zlib, and convert to string
print(zlib.decompress(sys.stdin.buffer.read()).decode())
```

- `sys.stdin.buffer.read()`: reads piped data as binary
- `zlib.decompress()`: decompresses zlib data to bytes
- `.decode()`: converts bytes to a UTF-8 string

## Summary

Flask session cookies are only signed, not encrypted, so the payload is readable by anyone.
This challenge delivers a simple but important lesson: never store sensitive information like flags in client-side sessions.
